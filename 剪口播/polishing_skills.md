# 口播文稿潤飾 Skill（手動維護）

> 本檔案由人工維護，不會被 autoresearch 自動改寫。
> 若要調整 AI 斷句/標點行為，請直接編輯這份文件。

## 核心原則

潤飾 = **只加標點 + 只做語意判斷**。**絕對不刪字、不改字、不合併句子。**
這個階段的產物要送到「剪輯階段」做語意重複判斷，所以保留原文完整性比「讀起來通順」更重要。

## 標點選用

| 標點 | 使用時機 |
|------|---------|
| `，` | 語氣停頓、子句切分、未完成的陳述、短語連接 |
| `。` | 完整陳述的結尾（主謂齊全且下一段有明顯停頓） |
| `？` | 明確疑問語調（「嗎/呢/吧」結尾、反問、選擇疑問） |
| `！` | 強烈情緒或命令語氣（**慎用**，不確定時用句號或逗號） |
| `、` | 並列名詞之間（蘋果、香蕉、橘子） |
| `：` | 引入解釋或列舉（答案：；包含：） |
| `；` | 平行長句之間（很少用） |

## 決策優先順序（由上往下套用）

1. 段落以疑問詞結尾（嗎/呢/何/誰/什麼/哪/如何/怎樣）→ **問號**
2. 段落是完整陳述句（主語 + 謂語 + 受詞）且下一段 gap ≥ 1s → **句號**
3. 段落是不完整短語，或下一段 gap < 1s → **逗號**
4. 多個名詞並列 → **頓號**
5. 不確定 → **逗號**（最安全）

## 語氣詞處理

- 「啊/呢/嗎/吧/喔/嗯/哦」**原字保留**，只在後面加標點
- 不要刪除語氣詞（那是剪輯階段的事）
- 句中語氣詞用逗號串接（例：「你看，啊，這個...」）

## 專有名詞保護

- 疑似術語、人名、品牌名、專有名詞 → **不拆、不加內部標點**
- 舉例：「LangChain」「GPT-4」「claude-sonnet-4-6」→ 整塊保留，前後才加標點
- 中英混用時，英文名詞前後可加逗號區隔（例：「這個，LangChain，很好用。」）

## 時間戳對齊鐵律（最重要）

- `displayText` 只能在原字元之間**插入標點**
- **不可增字、不可減字、不可替換字**
- Phrase 邊界由上游決定，潤飾階段不重劃
- 違反此規則會讓時間戳對齊爆炸，整條剪輯管線會錯位

## 反面範例（這些都不對）

| 錯誤 | 原因 |
|------|------|
| 原文「嗨我多米」→ 輸出「大家好，我是多米」 | 增字、改字 |
| 原文「呃那個那個所以說」→ 輸出「所以說」 | 刪字（那是剪輯的事） |
| 原文兩段「我覺得」「這個很重要」→ 合成一段「我覺得這個很重要。」 | 合併句子（破壞 phrase 邊界） |
| 原文「LangChain很好」→ 輸出「Lang，Chain，很好。」 | 拆專有名詞 |

## 正面範例

| 原文 | 潤飾後 |
|------|--------|
| 嗨 我多米 | 嗨，我多米， |
| Olint連結寫作積木 | Olint連結寫作積木， |
| 那Borange它是代表什麼呢 | 那Borange它是代表什麼呢？ |
| 這個 啊 我覺得喔 | 這個，啊，我覺得喔， |
| 第一是速度 第二是品質 | 第一是速度、第二是品質。 |
| 所以結論就是 寫作需要練習 | 所以結論就是，寫作需要練習。 |
