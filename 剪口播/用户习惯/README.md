## 这些偏好现在由谁执行（2026-07 现况）

> 旧的「AI 审核顺序（按优先级）」描述的是已退役的 legacy 引擎（auto_select_rules.js，
> 只剩训练层在用）。现行 8900 剪片主线是 phrase 级链：
> `ai_polish → phrase_prefilter → ai_cut_pairs → convert_ai_to_indices`（+ 重录/咳嗽预选桥接）。
> 各偏好对应的现行实现：

| 偏好 | 现行实现（phrase 级主线） |
|------|--------------------------|
| 静音分级（3） | phrase_prefilter 规则 A（gapAfter ≥ silence.threshold）+ 汇出时 refine_segments 停顿压平（吃音讯实测 silences.json，分气口/转场两级） |
| 重复句（4） | 规则 B 相邻前 5 字相同（留后删前，P1 念糊才翻盘）+ 规则 D take grouping + 候选对→AI 判决 |
| 卡顿词（5） | 规则 C 固定清单 + 规则 C2 通用短单元立即重复（2–20 字 AB AB 型，白名单挡叠词） |
| 句内重复（6） | 规则 G 句首长前缀 + 规则 C2（零间隔）+ 规则 C3 隔字重复（A+迟疑词+A，中间是内容词不动） |
| 连续语气词（7）| inline_filler_trim（句中孤立嗯/呃/欸/哦/噢；「啊」需前接标点） |
| 重说纠正（8） | detect_retakes exact（未校正 whisper_words，删前留后）+ fuzzy 近距/远距（隔 1–2 句碎片，审核页预选） |
| 残句（9） | 规则 F2 放弃句首（连接词开头+停顿+下句更长）+ fuzzy 远距层把碎片连同 false-start 一起删 |
| 保留连接词（10）| loadProtectedWords → 规则 F 跳过、validate_selection 校验 |
| 语意重复（11） | 候选对（bigram/LCS 门槛）→ ai_cut_pairs AI 判决 |

## 核心原则

**删前保后**：后说的通常更完整，删除前面保留后面。
（唯一例外：后段被 STT confidence 证实「念糊」才删后留前，见 rules/04。）
