## 這些偏好現在由誰執行（2026-07 現況）

> 舊的「AI 審核順序（按優先級）」描述的是已退役的 legacy 引擎（auto_select_rules.js，
> 只剩訓練層在用）。現行 8900 剪片主線是 phrase 級鏈：
> `ai_polish → phrase_prefilter → ai_cut_pairs → convert_ai_to_indices`（+ 重錄/咳嗽預選橋接）。
> 各偏好對應的現行實現：

| 偏好 | 現行實現（phrase 級主線） |
|------|--------------------------|
| 靜音分級（3） | phrase_prefilter 規則 A（gapAfter ≥ silence.threshold）+ 匯出時 refine_segments 停頓壓平（吃音訊實測 silences.json，分氣口/轉場兩級） |
| 重複句（4） | 規則 B 相鄰前 5 字相同（留後刪前，P1 念糊才翻盤）+ 規則 D take grouping + 候選對→AI 判決 |
| 卡頓詞（5） | 規則 C 固定清單 + 規則 C2 通用短單元立即重複（2–20 字 AB AB 型，白名單擋疊詞） |
| 句內重複（6） | 規則 G 句首長前綴 + 規則 C2（零間隔）+ 規則 C3 隔字重複（A+遲疑詞+A，中間是內容詞不動） |
| 連續語氣詞（7）| inline_filler_trim（句中孤立嗯/呃/欸/哦/噢；「啊」需前接標點） |
| 重說糾正（8） | detect_retakes exact（未校正 whisper_words，刪前留後）+ fuzzy 近距/遠距（隔 1–2 句碎片，審核頁預選） |
| 殘句（9） | 規則 F2 放棄句首（連接詞開頭+停頓+下句更長）+ fuzzy 遠距層把碎片連同 false-start 一起刪 |
| 保留連接詞（10）| loadProtectedWords → 規則 F 跳過、validate_selection 校驗 |
| 語意重複（11） | 候選對（bigram/LCS 門檻）→ ai_cut_pairs AI 判決 |

## 核心原則

**刪前保後**：後說的通常更完整，刪除前面保留後面。
（唯一例外：後段被 STT confidence 證實「念糊」才刪後留前，見 rules/04。）
