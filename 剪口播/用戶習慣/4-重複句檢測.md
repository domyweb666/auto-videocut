# 重複句檢測

## 定義

相鄰句子（被靜音分隔）開頭相同 ≥5字，通常是說錯後重說。

## 核心原則

**先分句，再比對**：必須先按靜音切分成句子列表，然後比對相鄰句子。

## 正確分析方法

```
✓ 正確：按靜音切分句子 → 比對相鄰句子開頭 → 整句刪除
✗ 錯誤：逐字掃描 → 發現重複片段 → 只刪片段
```

### 步驟

1. **按靜音切分句子**（靜音≥0.5s作為分隔）
2. **比對相鄰句子**（開頭≥5字相同 → 刪較短的整句）
3. **比對隔一句**（中間是殘句時，也要檢查前後句是否重複）

## 檢測邏輯

```javascript
// 相鄰句子比對
if (curr.text.slice(0, 5) === next.text.slice(0, 5)) {
  const shorter = curr.text.length <= next.text.length ? curr : next;
  markAsError(shorter);  // 刪除整句，不只是重複部分
}

// 隔一句比對（中間是短句/殘句時）
if (mid.text.length <= 5) {  // 中間是殘句
  if (curr.text.slice(0, 5) === next.text.slice(0, 5)) {
    markAsError(curr);   // 刪前句
    markAsError(mid);    // 刪殘句
  }
}
```

## 案例

| 前句 | 後句 | 刪除 |
|------|------|------|
| "這是我剪出來的一個案例" | "這是我剪出來的一個案例" | 前句（完全重複） |
| "我用cloud code的excuse功能做一個剪輯agent" | "所以我用cloud code的excuse功能做一個剪輯agent" | 前句 |
| "第二個是是q制技能系統第二個" | "第二個是scale技能系統" | 前句 |
| "好我們接下來開始怎麼去" | "好我們接下來開始怎麼去做一個剪口撥" | 前句 |
| "我們就可以看到這裡新的視頻" | "我們就可以看到這裡新的視頻了" | 前句 |

## 隔一句重複（中間有殘句）

當兩句之間隔了一個短殘句時，也要檢測：

```
句A: "這是我剪出來的一個案例"
殘句: "他呢"                    ← 中間隔了殘句
句B: "這是我剪出來的一個案例"

→ 刪 句A + 殘句
```

| 前句 | 中間殘句 | 後句 | 刪除 |
|------|---------|------|------|
| "這是我剪出來的一個案例" | "他呢" | "這是我剪出來的一個案例" | 前句 + 殘句 |
| "具體怎麼做呢我們首先下載這個" | "提示詞" | "具體怎麼做呢我們首先複製這個提示詞" | 前句 + 殘句 |
| "打開我們的AI" | "打開我們的" | "打開我們的AI然後告訴他去下載" | 前句 + 殘句 |

## 多次重複

連續說 3 次以上，刪除所有不完整的，保留最後完整的：

```
"以前呢我會把所有的技能"     → 刪
"以前呢我會把所有的功能做"   → 刪
"以前呢我會把所有的功能做都" → 刪
"以前呢我會把所有的功能都做成一個大的scale" → 保留
```

## 易錯點

```
❌ 逐字掃描，只發現局部重複片段
✓ 先分句，比對整句開頭，刪除整句

❌ 只比對相鄰句子
✓ 也要比對隔一句（中間可能是殘句）
```
